# Importing Data

Hands makes it easy to import data from files, APIs, or databases into your workbook's PostgreSQL database.

## Quick Import

### Drag and Drop

The fastest way to import is drag and drop:

1. Open your workbook
2. Drag a file onto the Hands window
3. The AI analyzes the file and creates a table

```
AI: I've imported sales_data.csv into the 'sales_data' table.

Created columns:
- id (integer, primary key)
- date (date)
- customer (text)
- amount (numeric)
- region (text)

Imported 10,847 rows.
```

### Using the Import Agent

For more control, use the `@import` agent:

```
@import /path/to/your/file.csv
```

The import agent:
- Analyzes file structure
- Infers optimal data types
- Creates the table with proper constraints
- Handles large files efficiently

## Import Options

### Custom Table Name

```
@import /path/to/file.csv as orders
```

### Append to Existing Table

```
@import /path/to/new_data.csv into orders
```

### Specify Data Types

```
Import this file with:
- date as timestamp
- amount as decimal(10,2)
- id as uuid
```

## Schema Inference

Hands automatically infers schemas from your data:

| Source Type | PostgreSQL Type |
|-------------|-----------------|
| `123` | `integer` |
| `123.45` | `numeric` |
| `"text"` | `text` |
| `true/false` | `boolean` |
| `2024-01-15` | `date` |
| `2024-01-15T10:30:00` | `timestamp` |
| `null` | Nullable column |

## Large File Handling

For files over 100MB, Hands uses streaming import:

```
@import /path/to/large_file.csv

AI: This is a large file (2.3 GB). I'll import it in batches.

Progress: [████████░░] 80% - 1,840,000 / 2,300,000 rows
```

The import runs in the background so you can continue working.

## API Data Import

Import data from REST APIs:

```
Fetch data from https://api.example.com/users and save to users table
```

For authenticated APIs:

```
Fetch data from https://api.example.com/orders
with header Authorization: Bearer YOUR_TOKEN
and save to orders table
```

## Database Import

Import from external databases:

```
Import the customers table from postgres://user:pass@host:5432/db
```

<Warning>
Never share database credentials in public conversations. Use environment variables for sensitive values.
</Warning>

## Scheduled Imports

For recurring data imports, create an integration:

```typescript
// integrations/sync-data.ts
import { integration } from "@hands/stdlib";

export const handler = integration(async (ctx) => {
  const response = await fetch("https://api.example.com/data");
  const data = await response.json();

  await ctx.sql`
    INSERT INTO api_data (payload, fetched_at)
    VALUES (${JSON.stringify(data)}, NOW())
  `;

  return { status: "ok", rows: data.length };
});
```

Configure the schedule in `hands.config.ts`:

```typescript
integrations: {
  "sync-data": {
    type: "polling",
    schedule: "rate(1 hour)",
    handler: "integrations/sync-data.handler"
  }
}
```

## Validating Imports

After importing, verify your data:

```
Show me the first 10 rows of the new table

How many rows were imported?

Are there any null values in required columns?

Check for duplicate entries
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Import seems stuck">
    For large files, check the background tasks indicator in the toolbar. The import continues even if you close the chat.
  </Accordion>

  <Accordion title="Wrong data types">
    Ask the AI to alter the column:
    ```
    Change the 'amount' column to decimal(10,2)
    ```
  </Accordion>

  <Accordion title="Encoding issues">
    Specify the encoding:
    ```
    @import file.csv with encoding 'latin1'
    ```
  </Accordion>

  <Accordion title="Date parsing errors">
    Specify the date format:
    ```
    @import file.csv with date format 'DD/MM/YYYY'
    ```
  </Accordion>
</AccordionGroup>
